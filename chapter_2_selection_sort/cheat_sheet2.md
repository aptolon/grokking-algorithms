 * Сортировка выбором имеет O(n^2) по скорости и O(n) по памяти

 * Цикл таков: ищем минимальное в arr1 -> добавляем в arr2 -> убираем минимальное из arr1 -> повторяем

 | Операция              | Массив (Array) | Связанный список (Linked List) |
|-----------------------|----------------|--------------------------------|
| Доступ по индексу     | O(1)           | O(n)                           |
| Поиск                 | O(n)           | O(n)                           |
| Вставка в начало      | O(n)           | O(1)                           |
| Вставка в конец       | O(1)*          | O(1)**                         |
| Вставка в середину    | O(n)           | O(n)                           |
| Удаление из начала    | O(n)           | O(1)                           |
| Удаление из конца     | O(1)*          | O(n)**                         |
| Удаление из середины  | O(n)           | O(n)                           |

O(1)* — для массива, если есть свободное место (без перераспределения памяти).
O(1)** — для связанного списка, если есть указатель на конец (tail).
В связном списке многие операции требуют предварительного поиска элемента → поэтому часто получается O(n).
